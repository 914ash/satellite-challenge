# Contributing

Keep this repository aligned with its public-safe mission profile.

- do not reintroduce challenge variants or local scratch folders
- do not commit provider credentials or local `.env` files
- keep screenshots, artifacts, and docs free of sensitive context
- preserve the canonical repo shape: `apps/`, `packages/`, `docs/`, and curated assets only
- use real screenshots in public docs; do not add synthetic cover/preview art
- keep README openings plain: what the app is and why it matters in two sentences
